# Правила работы с проектом Doc Orchestra

## Структура страниц

- Все страницы (родитель, кокпит, любые новые UI) делаем в **одном HTML-файле** — без внешних библиотек (никакого Bootstrap, Tailwind, React и т.д.)
- Стили пишем прямо в `<style>` внутри того же файла
- JS пишем прямо в `<script>` внутри того же файла
- Исключение: WebSocket и Fetch API — это встроено в браузер, их использовать можно

## Дизайн

- **Фон:** тёмный (примерно `#1a1a2e` или `#0f0f23`)
- **Текст:** светлый (`#e0e0e0` или `#ffffff`)
- **Акцентный цвет:** оранжевый (`#ff6b35` или `#ff8c42`)
- Никаких ярких разноцветных элементов — медицинский профессиональный вид
- Кнопки: оранжевые для основных действий, серые для второстепенных

## Адаптивная вёрстка

- Всё должно хорошо выглядеть на мобильных (телефон 375px и выше)
- Используем CSS Flexbox или Grid — без фиксированных пикселей для ширины
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — обязательно

## Стиль текста (в UI и в промптах агентов)

- Простой и дружелюбный — пишем как человек, не как инструкция
- Без канцелярита: не «осуществить», а «сделать»; не «в данный момент», а «сейчас»
- Сообщения ассистента родителям — тёплые, без медицинского жаргона
- Сообщения в кокпите врача — чёткие и по делу, минимум слов

## Перед созданием нового файла

- Сначала объяснить что планируется сделать и зачем
- Дождаться подтверждения от пользователя
- Только потом создавать файл

## Безопасность (нельзя нарушать)

- `.env` никогда не коммитить в git
- Guardrail в `orchestrator/_finalize_and_send()` не трогать без явного запроса
- API ключи только через переменные окружения — никогда в коде напрямую
