## 🔴 КРИТИЧЕСКОЕ: JS в кокпите полностью не работает

**Симптомы:** кокпит открывается, но кнопки мёртвые, карточки не кликаются, очередь пустая даже при активных сессиях.

**Причина:** синтаксическая ошибка в JavaScript внутри Python `"""..."""` строки.
Внутри тройных кавычек Python `\'` превращается в `'` (обычную кавычку), а не в экранированную. Одна такая ошибка в любом месте `<script>` блока — и **весь JS перестаёт парситься браузером**, весь блок скрипта игнорируется.

**Как поймать:**
```bash
python3 -c "
with open('mvp.py') as f: src = f.read()
start = src.find('COCKPIT_HTML = \"\"\"')
end = src.find('\"\"\"', start + 17)
html = src[start+17:end]
js = html[html.rfind('<script>')+8:html.rfind('</script>')]
open('/tmp/cockpit_check.js','w').write(js)
" && node --check /tmp/cockpit_check.js
```

**Правило:** при добавлении HTML с `onclick` атрибутами — никогда не вставлять одиночные кавычки напрямую. Использовать `data-id` атрибуты:
```python
# ❌ СЛОМАЕТ JS:
'<button onclick=\'myFunc("' + id + '")\'>...'

# ✅ ПРАВИЛЬНО:
'<button onclick="myFunc(this.dataset.id)" data-id="' + escapeHtml(id) + '">...'
```

## 🟡 Кокпит показывает старую версию страницы (кэш браузера)

**Симптомы:** после перезапуска сервера кокпит выглядит как раньше, новые функции не появляются. Особенно часто в Safari.

**Причина:** браузер закэшировал HTML страницу кокпита.

**Решение:** сервер автоматически добавляет `?v=SERVER_START` к URL кокпита при каждом запуске — это сбрасывает кэш. Если браузер всё равно показывает старое:
1. Сделай **hard refresh**: `Cmd+Shift+R` (Mac) или `Ctrl+Shift+R` (Windows)
2. Или открой кокпит в режиме инкогнито
3. Или вручную добавь `?v=` с любым числом в конец URL

## 🟡 Кокпит не открывался в Safari (HTTP Basic Auth)

**Статус:** исправлено, не вернётся.

**Что было:** авторизация через HTTP Basic Auth (`WWW-Authenticate: Basic`) — Safari блокировал её на localhost.

**Решение:** заменили на HTML-форму + cookie (`cockpit_session`, SameSite=lax, HttpOnly).

## 🟡 WebSocket не подключается в Safari (жёлтый индикатор)

**Симптомы:** в шапке кокпита индикатор соединения остаётся жёлтым.

**Причина:** Safari блокирует `ws://localhost` WebSocket соединения.

**Решение:** основной канал данных — SSE (`/api/events`), WebSocket оставлен как legacy. REST polling каждые 5 сек как запасной вариант. Жёлтый индикатор в Safari — норма, всё работает через SSE.

## 🟡 Очередь кокпита пуста после перезапуска сервера

**Симптомы:** после изменений в коде и перезапуска сервера все карточки исчезают.

**Причина:** сессии хранятся в памяти (`dict sessions` в mvp.py). При перезапуске Python процесса — всё стирается. Это не баг, это текущая архитектура.

**Решение:** создать новую сессию через родительский чат:
1. Открыть `http://localhost:8081/parent`
2. Написать несколько сообщений ассистенту
3. Карточка появится в кокпите автоматически

**Долгосрочное решение:** миграция на PostgreSQL запланирована в Фазе 3.

## 🟡 Временная шкала календаря сливалась с фоном

**Статус:** исправлено в коммите `5e77a30`.

**Причина:** цвет текста `#2a2a4a` на фоне `#08080f` — почти идентичны.

**Решение:** поднят до `#6b6b9a`, разделитель до `#2a2a4a`.

## 🟡 Текст в интерфейсе плохо читался (низкий контраст)

**Статус:** исправлено в коммите `e4e9ef8`.

**Причина:** цвета `#444`, `#555`, `#666` на тёмном фоне `#0f0f23` / `#1a1a2e` почти невидимы.

**Решение:** все вторичные тексты подняты до `#6666aa–#aaaacc` (фиолетово-серая палитра).

**Правило на будущее:** минимально допустимый цвет текста на нашем тёмном фоне — `#666688`. Всё темнее — использовать только для декоративных элементов, не для читаемого текста.

## 🟢 Как проверить что всё в порядке перед запуском

```bash
# 1. Python синтаксис
python3 -c "import ast; ast.parse(open('mvp.py').read()); print('Python: OK')"

# 2. JS синтаксис (извлекаем из HTML шаблона)
python3 -c "
with open('mvp.py') as f: src = f.read()
start = src.find('COCKPIT_HTML = \"\"\"')
end = src.find('\"\"\"', start + 17)
html = src[start+17:end]
js = html[html.rfind('<script>')+8:html.rfind('</script>')]
open('/tmp/cockpit_check.js','w').write(js)
" && node --check /tmp/cockpit_check.js

# 3. Запуск сервера
.venv/bin/python3 mvp.py

# 4. Проверка что сервер поднялся
curl -s -o /dev/null -w "%{http_code}" http://localhost:8081/login
# Должно вернуть 200
```
