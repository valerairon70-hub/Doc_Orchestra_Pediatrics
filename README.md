# Doc Orchestra — Pediatrics

**AI-ассистент для врача педиатра-инфекциониста**

Многоагентная система, которая автоматизирует рутинную коммуникацию с родителями пациентов: собирает жалобы через диалог, формирует SOAP-заметки и подаёт готовые ответы врачу на одно нажатие. Врач видит структурированную карточку и одобряет ответ — вся переписка проходит через него.

---

## Быстрый старт

### 1. Клонировать и установить зависимости

```bash
git clone https://github.com/valerairon70-hub/Doc_Orchestra_Pediatrics.git
cd Doc_Orchestra_Pediatrics
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Создать файл с ключами

```bash
cp .env.example .env
```

Открыть `.env` и вставить:
```
ANTHROPIC_API_KEY=sk-ant-...  # обязательно
COCKPIT_PASSWORD=doctor123    # пароль кокпита врача
```

### 3. Запустить MVP

```bash
.venv/bin/python3 mvp.py
```

### 4. Открыть в браузере

| Роль | URL |
|------|-----|
| Родитель (чат с ассистентом) | http://localhost:8081/parent |
| Врач (кокпит одобрения) | http://localhost:8081/cockpit |

## Как это работает

```
Родитель пишет жалобу
       ↓
AI-ассистент (Claude Haiku) ведёт диалог, собирает симптомы
       ↓
Генерируется SOAP-заметка + черновик ответа
       ↓
Врач видит карточку в кокпите → нажимает «Одобрить»
       ↓
Ответ уходит родителю
```

**Ни одно сообщение не уходит родителю без явного одобрения врача** — это жёсткое правило, зашитое в код.

## Что работает сейчас (MVP)

- Диалог с родителем — AI задаёт уточняющие вопросы (симптомы, температура, возраст, длительность)
- Экстренный режим — «ребёнок задыхается» → мгновенный ответ «103!» без вызова API
- SOAP-заметка — автоматически после сбора жалоб
- Кокпит врача — очередь запросов, SOAP, диалог, вкладка интерпретации анализов
- Шаблоны быстрых ответов — 6 кнопок (жаропонижающее, скорая, наблюдение и др.)
- Расписание приёмов — календарь на неделю в кокпите
- Интерпретация анализов — врач вставляет ОАК/рентген, ИИ разбирает в контексте жалоб
- Telegram-уведомления врачу — при новом запросе (настраивается в `.env`)
- Автоответ вне рабочих часов — «врач ответит утром» при ночных обращениях
- Авторизация кокпита — HTML-форма + cookie

## Переменные окружения

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `ANTHROPIC_API_KEY` | Да | Ключ Claude API |
| `COCKPIT_PASSWORD` | Да | Пароль входа в кокпит |
| `DOCTOR_NAME` | Нет | Имя врача в приветствии (пр. «доктора Иванова») |
| `TELEGRAM_BOT_TOKEN` | Нет | Уведомления врачу в Telegram |
| `DOCTOR_CHAT_ID` | Нет | Telegram chat_id врача |
| `WORK_START` / `WORK_END` | Нет | Рабочие часы (по умолчанию 9–20) |
| `REMINDER_TIMEOUT_MIN` | Нет | Напоминание если не одобрено (по умолчанию 30 мин) |

## Структура проекта

```
mvp.py                  ← Запускаемый MVP (FastAPI, порт 8081) — основной файл
doc_orchestra.py        ← Полная версия (требует Gemini API)

agents/                 ← AI-агенты (в MVP не активны, готовы к интеграции)

  vision_agent.py       — анализ фото и PDF
  mx_agent.py           — план лечения по NICE/BMJ

core/                   ← Схемы, конфиг, база данных, очереди
tools/                  ← Калькулятор доз, z-score ВОЗ, PDF-парсер
data/guidelines/        ← Гайдлайны NICE (лихорадка, анемия)

PLAN.md                 ← Дорожная карта (Фазы 0–8)
project.md              ← Архитектура и описание
KNOWN_ISSUES.md         ← Известные проблемы и решения
CLAUDE.md               ← Правила для AI-ассистента (Claude Code)
```
