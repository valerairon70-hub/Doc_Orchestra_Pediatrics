## 1. Аналоги — 5 похожих приложений

### 1. Klara (США)
**Сайт:** klara.com
**Для кого:** частные клиники и групповые практики, США
**Что умеет:**
- Двусторонние SMS/чат с пациентами (HIPAA-compliant)
- Автоматические напоминания о приёмах
- Видео-консультации (телемедицина)
- Внутренний чат между сотрудниками
- Интеграция с EHR (Electronic Health Records)
- Автоматизация: сбор анамнеза до приёма через анкеты
- Групповые рассылки пациентам

**Хорошее:** G2 признал лучшим по ROI среди HIPAA-мессенджеров (2024). Удобный интерфейс, не перегружен. Врачи хвалят что снизило количество звонков на 40-60%.

**Плохое:** Цена скрытая (quote-based), по отзывам от $500/мес для малой практики. Нет AI — только правила и автоматизации. Слабая аналитика. Нет поддержки мессенджеров (только SMS/web-chat).

**AI-компонент:** Нет. Только правила и шаблоны.

### 2. Luma Health (США)
**Сайт:** lumahealth.io
**Для кого:** клиники любого размера, акцент на операционную эффективность
**Что умеет:**
- Самозапись пациентов онлайн
- Автоматические напоминания (SMS, email, WhatsApp)
- Двусторонний чат
- Сбор отзывов и NPS после визита
- Интеграция с EHR
- Заполнение форм и согласий онлайн до приёма
- Управление листом ожидания

**Хорошее:** По их данным, Operational AI сэкономил 2 млн часов персонала в 2025. Сильная автоматизация логистики (напоминания, записи). Реально снижает no-show на 30-40%.

**Плохое:** Сложно настроить. Дорогой для solo-практики. AI в основном для операционной логистики, не для медицинского диалога. Много жалоб на техподдержку.

**AI-компонент:** Operational AI для оптимизации расписания и снижения пропусков. Не медицинский диалог.

### 3. Doximity (США)
**Сайт:** doximity.com
**Для кого:** врачи (более 80% врачей США зарегистрированы), профессиональная сеть
**Что умеет:**
- Защищённый мессенджер врач-врач
- Телефонные звонки с маскировкой номера
- Видео-консультации
- Профессиональная сеть ("LinkedIn для врачей")
- Новостная лента по специальности
- Электронные рефералы между врачами

**Хорошее:** Базовый план бесплатен. Массовое проникновение в США — врачи уже там. Удобно для консультаций с коллегами и реферального потока.

**Плохое:** Ориентирован на коммуникацию врач-врач, не врач-пациент. Нет AI для сбора жалоб. Монетизирует через рекламу фармкомпаний — это вызывает этические вопросы.

**AI-компонент:** Минимальный. Недавно добавили AI для транскрипции и черновиков документов.

### 4. Ada Health (Германия/США)
**Сайт:** ada.com
**Для кого:** пациенты для самодиагностики + B2B для страховых и клиник
**Что умеет:**
- AI-симптом-чекер: задаёт вопросы и предлагает возможные диагнозы
- 13+ млн активных пользователей, 20+ млн оценок состояния
- 11 языков
- B2B API для интеграции в другие сервисы
- Направление к специалисту по итогам оценки

**Хорошее:** Точность ~70% (против 82% у GP — неплохо для AI). Доказательная медицина в основе. Хороший UX для пациентов — не пугает, объясняет доступно.

**Плохое:** Нет функций для врача — нет кокпита, нет очереди, нет одобрения. Ориентирован на самодиагностику, не на коммуникацию. Высокий процент тревожных пользователей.

**AI-компонент:** Ключевой — весь продукт построен на нём. Модель обучена на медицинской литературе.

### 5. СберЗдоровье (Россия, бывший DocDoc)
**Сайт:** sberhealth.ru
**Для кого:** пациенты в России ищущие врача, врачи расширяющие практику
**Что умеет:**
- Маркетплейс врачей (40+ специальностей)
- Онлайн-запись на приём
- Видео и чат-консультации
- Интеграция с экосистемой Сбербанка
- Анализы и диагностика онлайн
- Психологи и ветеринары тоже есть

**Хорошее:** Огромная аудитория, доверие бренда Сбербанка. Пациентам бесплатно. Самая большая база врачей в России. Врачи получают новых пациентов без маркетинга.

**Плохое:** Маркетплейс, а не инструмент для врача. Врач получает пациентов, но дальше работает в своей системе. Нет AI-помощника для диалога. Высокая комиссия с врача за пациентов. Не подходит для solo-практики с постоянными пациентами.

**AI-компонент:** Минимальный — рекомендации врачей, базовая навигация.

## 3. UX и сценарии использования

### Сценарий 1 — Первичное обращение (22:30, мама с больным ребёнком)

**Флоу:**
1. Мама открывает ссылку (или пишет в Telegram)
2. Видит дружелюбный чат, быстро понимает что нужно описать проблему
3. Пишет: *"Сынок заболел, температура 38 и сыпь на животе"*
4. AI задаёт 2-3 уточняющих вопроса (как давно, возраст, есть ли ещё симптомы)
5. После ответов — сообщение "Передала врачу, он ответит утром. При ухудшении — 103"
6. Мама чувствует что её услышали — засыпает спокойнее

**Что работает хорошо:** Ощущение присутствия ("кто-то есть на связи"), структурированные вопросы экономят время, чёткий финал диалога.

**Что раздражает в плохих приложениях:**
- Бесконечные формы вместо чата
- Робот, который не понимает контекст и переспрашивает одно и то же
- Нет подтверждения что сообщение дошло
- Нет информации когда ждать ответа

---

### Сценарий 2 — Работа врача с очередью (8:00, начало рабочего дня)

**Флоу:**
1. Врач открывает кокпит на телефоне или ноутбуке
2. Видит 4 запроса за ночь — у каждого SOAP-заметка уже готова
3. Сортирует по срочности (сначала красные флаги)
4. Читает SOAP → правит черновик ответа → нажимает "Одобрить" (30 сек на кейс)
5. Один кейс требует звонка — звонит напрямую, помечает в системе
6. Все 4 кейса обработаны за 10 минут вместо 40

**Что работает хорошо:** Вся информация собрана до врача, черновик уже написан, нет лишних кликов.

**Что раздражает в плохих приложениях:**
- Нужно читать весь длинный диалог чтобы понять суть
- Нет черновика ответа — надо писать с нуля
- Кнопка "Одобрить" требует подтверждения ещё раз и ещё раз
- Интерфейс ломается на мобильном

---

### Сценарий 3 — Экстренный случай (3:00 ночи)

**Флоу:**
1. Папа пишет: *"Ребёнок задыхается"*
2. Система мгновенно (без API, без задержки): *"Немедленно вызовите 103! Уложите ребёнка на бок, не оставляйте одного"*
3. Кокпит помечает кейс 🚨 ЭКСТРЕННО
4. Врач получает уведомление на телефон немедленно (Telegram/SMS)

**Что критично:** Скорость и надёжность. Любая задержка недопустима. Должно работать даже при сбое AI-сервиса.

---

### Типичные UX-ошибки разработчиков

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Один интерфейс для врача и пациента | Разные задачи, разные контексты | Отдельные страницы с разным дизайном |
| Форма вместо чата | Пациент не понимает что заполнять | Диалог: один вопрос за раз |
| Нет подтверждения что сообщение дошло | Тревога: "А врач вообще получил?" | Явный статус "Передано врачу" |
| Меню-бургер на мобильном | Врач работает одной рукой | Главные действия всегда видны |
| Сложная авторизация для пациента | Пациент уходит не записавшись | Ссылка без логина или OTP |
| Жаргон в интерфейсе | Пациент не понимает "анамнез" | Простые слова: "Как давно болеет?" |
| Нет мобильной версии для врача | Врач часто на ходу | Mobile-first для кокпита |

## 4. Технологии — сравнение вариантов

### База данных

| | SQLite | PostgreSQL | MongoDB |
|---|--------|-----------|---------|
| **Когда** | MVP, прототип, 1 сервер | Production, рост, несколько врачей | Неструктурированные данные |
| **Плюсы** | Нулевая настройка, файл = база | ACID, конкурентный доступ, аудит-лог | Гибкость схемы |
| **Минусы** | Теряет данные при падении сервера, нет параллельных записей | Нужен отдельный сервер, сложнее | Слабые транзакции |
| **Цена** | Бесплатно | Бесплатно (self-hosted) / $15-25/мес (managed) | $0-57/мес |
| **Для Doc Orchestra** | ✅ Сейчас (MVP) | ✅ Следующий шаг (Фаза 6) | ❌ Не нужно |

**Вывод:** SQLite для MVP — ок. Переход на PostgreSQL нужен до первого реального врача.

## 5. Монетизация

### Как зарабатывают на инструментах для врачей

#### Модель 1 — Подписка (SaaS per seat)
Врач платит фиксированно в месяц. Самая распространённая и предсказуемая модель.

**Реальные цены на рынке:**
- SimplePractice (США, психотерапия): $49/мес — solo-план
- Kareo (США, общая медицина): $150+/мес
- Медицинские CRM в России: 3 000–15 000₽/мес
- Telegram-боты для врачей (РФ, самодельные): 1 000–5 000₽/мес

**Для Doc Orchestra:** Реалистичная цена для solo-педиатра — **2 000–5 000₽/мес** (≈$20-55). При 10 врачах — 20 000-50 000₽/мес.

#### Модель 2 — Freemium
Бесплатно для 1 врача / X запросов в месяц, платно за расширенные функции.

**Когда работает:** Для быстрого набора пользователей. Врачи консервативны — сначала хотят попробовать бесплатно.

**Риск:** "Халявщики" которые никогда не платят. Для solo-практики часто достаточно free-tier.

**Для Doc Orchestra:** Бесплатно для 1 врача до 50 запросов/мес → идеальный hook для демо.

#### Модель 3 — Pay-per-use
Платишь за количество обработанных запросов/пациентов.

**Пример:** Zocdoc берёт деньги за каждую новую запись ($35-80 за пациента в 2024).

**Для Doc Orchestra:** Можно брать ₽ за каждый одобренный ответ врача (≈ 50-100₽/запрос). При 20 запросах в день — 30 000₽/мес с одного врача.

#### Модель 4 — White-label для клиник
Продаёшь не врачу, а клинике/сети. Они брендируют под себя.

**Цены:** $500-5000/мес для клиники (10-100x больше чем solo).

**Риск:** Долгий цикл продажи (6-12 месяцев), нужны ресурсы на интеграцию.

**Для Doc Orchestra:** Хорошая цель для Фазы 4-5.

---

### Что НЕ работает для врачей

| Ошибка | Почему |
|--------|--------|
| Реклама фармкомпаний в инструменте врача | Разрушает доверие, этические вопросы |
| Комиссия с каждой консультации | Врачи воспринимают это как посягательство на доход |
| Дорогой онбординг (>$500) | Solo-врачи не готовы к авансу без проверки |
| Годовой контракт сразу | Нужно помесячно — врачи хотят выйти если не нравится |
| Скрытые цены | Врачи не любят звонить чтобы узнать стоимость — уходят к конкурентам |

### Российский рынок — особенности

**Конкуренты в РФ:**
- **СберЗдоровье** — маркетплейс, не инструмент врача
- **Яндекс.Здоровье** — телемедицина, видео-консультации
- **DocDoc** (часть Сбера) — поиск врача
- **MedMe.ru** — запись к врачу онлайн

**Ключевые отличия рынка:**
- Врачи привыкли к WhatsApp/Telegram как основному каналу — низкий барьер для интеграции
- Нет массовой EMR/EHR культуры → нет привычки платить за ПО
- Средний чек за инструмент: 1 000–5 000₽/мес приемлемо для solo-врача
- Важен вопрос хранения данных пациентов (152-ФЗ о персональных данных)

**Рекомендуемый путь для Doc Orchestra:**
1. **Сейчас:** Бесплатно для первого врача-партнёра → получить обратную связь
2. **3-6 месяцев:** 2 000₽/мес для ранних пользователей (early bird)
3. **6-12 месяцев:** 5 000₽/мес для solo, 15 000₽/мес для клиники (2-5 врачей)
4. **Год+:** White-label для педиатрических сетей

## Итоги и приоритеты для Doc Orchestra

### Что взять у конкурентов прямо сейчас
1. **Уведомления врача через Telegram** — это то, что есть у всех и чего нет у нас
2. **Персистентность данных** — все конкуренты хранят историю, мы теряем при рестарте
3. **Режим недоступности врача** — стандарт в Klara и Luma Health

### Уникальные преимущества Doc Orchestra (не нужно менять)
- **Guardrail** — ни один конкурент не требует одобрения каждого ответа AI
- **SOAP-заметка авто** — только Doximity пробует, у остальных нет
- **Экстренный режим без API** — нигде не видели

### Следующий шаг после демо с врачом
Перед любой монетизацией — пройти Фазу 2: показать врачу, получить обратную связь, дополнить промпты реальными кейсами.
